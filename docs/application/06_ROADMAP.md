# 06. 개발 로드맵 + 수정 사항 통합 ★

> **이 파일이 "기존 앱 + 앞으로 수정/추가할 내용"을 시점별로 분리해 통합 표기하는 핵심 문서입니다.**
> 심사위원은 이 파일 하나로 "지금 무엇이 있고, 앞으로 무엇을 만들 것인가"를 즉시 판별할 수 있습니다.

---

## 0. 표기 규약

| 표기 | 의미 | 검증 |
|------|------|------|
| 🟢 | **현재 구현 완료** (코드 존재, MVP에서 동작) | `파일:라인` 인용 동반 |
| 🟡 | **부분 구현** (UI만 또는 로컬만 — 보완 필요) | 갭과 보완 방향 명시 |

## 3. 🔵 Phase 2 — 지원 기간 내 신규 개발 (M+0~9)

예비창업패키지 약정 기간 동안 완료 목표.

### 3.1 사용자 인증·계정
- 이메일/카카오 OAuth 로그인
- 사용자 프로필 서버 저장
- 향후 모든 기능의 기반 (DB·푸시·가족 연동)

### 3.2 백엔드 DB 도입
- **선택지**: Supabase (PostgreSQL + Auth + Realtime) 또는 Firebase
- **저장 대상**:
  - 사용자 프로필·걸음 상태·즐겨찾기
  - 커뮤니티 제보 (localStorage → 서버)
  - GPS 트립 기록 (월간 통계용)
  - 가족 연동 토큰
- **기존 코드 영향**: `reportsStore.js` API 호환 유지하며 백엔드만 교체

### 3.3 실시간 가족 연동
- FCM (Firebase Cloud Messaging) 푸시 알림
- 가족 SOS 수신 → 실시간 위치 공유
- 도착 알림·체류 시작 알림
- WebSocket 또는 Realtime DB

### 3.4 PWA 전환
- `manifest.json` + `apple-touch-icon` + `theme-color`
- 홈 화면 추가 가능
- Service Worker
  - 지도 타일·POI 응답 캐시
  - 오프라인 기본 화면 (마지막 경로)

### 3.5 data.go.kr 전국 확대
- 현재: 5,000건 모듈 캐시 (한 덩어리)
- 개선: **지역별 캐시 분리** (서울/경기/광역시별)
- TTL 24h → 1주 (행정안전부 갱신 주기 고려)
- Cron 자동 갱신 (`vercel.json` cron)

### 3.6 접근성 보강 (WCAG 2.2 AA 준수)
- 글씨 크기 3단계 실제 적용 (🟡1 완료)
- **고대비 모드** (배경·텍스트 4.5:1 이상)
- **스크린리더 ARIA 라벨** 전체 페이지
- **키보드 네비게이션** (Tab 순서)
- **포커스 인디케이터** 명확화
- **접근성 외부 감수** (한국시각장애인연합회·접근성 인증)

### 3.7 베타 측정·운영 도구
- 익명 분석 (Plausible 또는 자체)
- 외출 횟수·거리·체류 시간 집계
- A/B 테스트 (음성 안내 톤·속도)
- 오류 보고 (Sentry)

## 4. ⚪ Phase 3 — 장기 확장 (M+9 이후)

### 4.1 네이티브 앱 (React Native)
- iOS / Android 백그라운드 GPS
- 푸시 알림 안정성 강화
- 배터리 최적화

### 4.2 ML 학습
- 사용자별 평균 걸음 속도 학습
- 자주 가는 곳 패턴 → 다음 외출 예측
- 시간대별 혼잡도 학습 (베타 데이터 누적 후)

### 4.3 가족 실시간 위치 공유
- 보호자 앱에서 부모/배우자 위치 지도
- 안전 구역 설정·이탈 알림
- 외출 히스토리 공유

### 4.4 다언어
- 영어·중국어·일본어
- 관광객 무장애 안내 시장
- 자치단체 외국인 정책 연계

### 4.5 B2B 관리자 대시보드
- 자치구·복지관용 무장애 데이터 환류
- 익명 사용자 통계 (외출 패턴·문제 지점)
- 정책 결정 지원

### 4.6 모빌리티 연동
- 시니어 콜택시 (카카오T 시니어 / 카카오모빌리티 API)
- 공유 전동스쿠터 (실버 모델)
- 휠체어 리프트 차량 호출

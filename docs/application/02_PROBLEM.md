# 02. 문제 인식 (Problem)

> 예비창업패키지 양식 매핑: **문제 인식 — 외부환경 분석 / 창업 동기 / 목표 시장**

---

## 1. 외부환경 분석

### 1.1 인구 구조 변화: 보행 약자 1,400만 명

| 그룹 | 인구 규모 | 출처 | 추세 |
|------|-----------|------|------|
| 65세 이상 고령자 | 약 **950만 명** (2024) | 행정안전부 주민등록인구통계 | 2030년 25% 진입 → **약 1,300만 명** |
| 등록 장애인 | 약 **264만 명** (2023) | 보건복지부 등록장애인 현황 | 지체장애 44% 비중 |
| 영유아 동반 보호자 | 약 **230만 가구** | 통계청 출생아·가구 추계 | 도시 거주 비율 90%↑ |

> 중복 제외 시 **국내 약 1,400만 명**이 일상 외출에서 보행 보조가 필요한 상태입니다.

### 1.2 도시 인프라의 한계

- **계단·턱·경사**: 서울시 보도 중 무장애 인증 비율 낮음 (자치구 편차 큼)
- **쉼터 부재**: 무더위쉼터는 여름 한정 운영, 일반 쉼터 표준 데이터 미통합
- **정보 분산**: 지하철 엘리베이터·장애인 화장실·경사로 정보가 각각 다른 시스템에 산재
- **실시간성 부족**: 공사·통행불가 정보가 보행 약자에게 도달하지 않음

### 1.3 정책·기술 환경

- 「**교통약자의 이동편의 증진법**」 시행 (2005~), 이동권 보장 의무화
- **행안부 무장애 생활환경 인증제도** (BF 인증) 확대
- **공공데이터 개방** 가속화 (data.go.kr ODCloud — 무더위쉼터·공중화장실·장애인편의시설 표준데이터)
- **AI 음성·지도 API** 일반화 → 기술적 진입장벽 낮아짐

## 2. 창업 동기 (스토리)

### 2.1 사용자가 외출을 포기하는 3가지 이유

1. **"오늘 갈 길에 계단이 몇 개인지 모른다"** — 경로 선택의 불확실성
2. **"중간에 쉴 곳이 있는지 모른다"** — 체력 한계 대비 불가
3. **"넘어지면 가족이 알 수 있을까"** — 안전망의 부재

### 2.2 기존 지도 앱의 구조적 한계

| 한계 | 원인 |
|------|------|
| **시각 중심 UI** | 운전자·통근자 대상 설계 — 글씨·아이콘 작음 |
| **자동차 경로 기반** | "도보" 경로도 차도 기준 알고리즘 사용 |
| **무장애 POI 없음** | 상업 POI 중심, 쉼터·장애인 화장실 미수록 |
| **안전 기능 없음** | 체류 감지·가족 알림 부재 |
| **공사·통행불가 미반영** | 실시간 커뮤니티 입력 채널 없음 |

### 2.3 우리의 가설

> "보행 약자가 외출을 포기하지 않게 하려면, **눈을 떼지 않고 듣기만 해도 안전한 길**과 **쉴 곳**, 그리고 **가족이 알 수 있는 안심망**을 한 앱에 묶어야 한다."

이 가설을 14개 페이지의 MVP로 검증 가능한 상태까지 만든 것이 현재 단계입니다.

## 3. 목표 시장 — 1차 타깃 페르소나 4종

코드의 4종 걸음 상태(`src/hooks/useAppState.jsx:22-55`)가 그대로 4 페르소나에 대응합니다.

### P1. "조금 느려요" (slow, 0.7~0.9 m/s)
- **대표 사용자**: 65~74세 활동적 고령자 (예: 시장·병원·복지관 정기 방문)
- **외출 빈도**: 주 3-5회
- **핵심 니즈**: 가까운 화장실·휴게 의자 자동 안내
- **현재 제품 대응**: `WalkState.jsx`, `Home.jsx` 빠른 찾기, `Resting.jsx` 자동 휴식

### P2. "많이 느려요" (very-slow, 0.5~0.7 m/s)
- **대표 사용자**: 75세 이상 / 지팡이 사용자
- **외출 빈도**: 주 1-2회
- **핵심 니즈**: 짧은 거리·자주 쉼·계단 회피
- **현재 제품 대응**: 50m 반경 쉼터 필터링(`RouteSuggest.jsx:150-164`), 체류 감지(`useGPS.js:70-92`)

### P3. "도움이 필요해요" (needs-help, ≤0.4 m/s)
- **대표 사용자**: 휠체어·보행보조기·시각장애인
- **외출 빈도**: 주 1회 또는 동반자 동반
- **핵심 니즈**: **무장애 경로 절대 보장** + 음성 양방향 + 가족 SOS
- **현재 제품 대응**: 도보 경로 폴백(`api/route.js`), 음성 안내, `SOS.jsx` 3단계

### P4. "유모차예요" (stroller, 0.6~0.8 m/s)
- **대표 사용자**: 영유아 동반 보호자 (특히 도심)
- **외출 빈도**: 주 5회 이상
- **핵심 니즈**: 경사로·엘리베이터·기저귀실
- **현재 제품 대응**: 경사로·엘리베이터 POI 마커(`useKakaoMap.js:70-155`)

### 세컨더리 사용자: 보호자(가족)
- 위 P1~P4 사용자의 자녀·배우자
- 실시간 위치 확인, 도착 알림, SOS 수신
- 현재 제품 대응: `Family.jsx`, `SOS.jsx` (서버 연동은 Phase 2에서 완성)

## 4. 시장 트렌드: 지금이 적기인 이유

### 4.1 인구 구조 (Pull 요인)
- 2030년 65세↑ 25% 진입 → 보행 약자 시장 급팽창
- 1인 고령가구 증가 → 자녀가 원격으로 챙겨야 할 부모 증가

### 4.2 정책 (Push 요인)
- 서울시 **'보행친화도시 서울'** 비전 2030
- 행안부 **'무장애 생활환경 조성'** 의무화 확대
- 보건복지부 **장애인 이동권 강화** 정책

### 4.3 기술 (Enabler)
- Kakao Local · Tmap · OSRM 등 무료/저비용 지도 API 성숙
- Web Speech API 안정화 → 음성 기능 클라우드 비용 ≈ 0
- data.go.kr 표준 데이터셋 5000건 단위로 일괄 다운로드 가능

> "기술이 가능해졌고, 인구가 필요로 하며, 정책이 지원합니다. **2026~2028년이 진입 적기**입니다."
